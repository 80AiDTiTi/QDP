# QDP 達爾飛法風險評估系統

> 基於 Google Apps Script 的多單位、多活動風險評估平台。透過達爾飛法（Delphi Method）流程，以匿名投票方式收斂各部門意見，產出優先排序的風險對策表。

## 系統簡介

QDP 系統以 Google Sheets 作為資料庫、Google Apps Script 作為後端、HTML/CSS/JS 作為前端，無需任何伺服器或外部服務，即可在組織內部快速部署。

**適用情境：** 年度風險評估、ISO 管理審查、跨部門議題蒐集與優先排序。

---

## 達爾飛法流程

```
步驟一：列舉要因          → 各部門匿名提出關注議題
步驟二：共同討論管理重點   → 管理員設定投票規則與票數
步驟三：私下計分           → 參與人員匿名對各議題評分
步驟四：匯總分數           → 系統自動統計總分 / 票數
步驟五：排列優先順序       → 管理員依總分排定對策優先次序
```

---

## 功能特色

- **多活動管理**：可同時建立多個評估活動，各自獨立運作
- **四階段流程控制**：管理員可切換各活動的開放階段（提案 → 投票 → 評估對策）
- **匿名投票機制**：投票人以 `#1`、`#2` 代號顯示，前端不顯示真實姓名
- **並發保護**：後端使用 `LockService` 防止多人同時投票造成匿名代號衝突
- **權限分離**：一般人員為唯讀模式，僅管理員可編輯評估紀錄與對策表
- **CSV 匯出**：評估紀錄表與對策表均支援匯出，相容 Excel（含 UTF-8 BOM）
- **RWD 支援**：支援手機與桌機介面，側邊欄可收合

## 部署方式

### 前置需求

- Google 帳號
- 可存取 Google Sheets 與 Google Apps Script

### 步驟

**1. 複製 Google Sheets 試算表**

https://docs.google.com/spreadsheets/d/1DOBDnBnNmSdT08d8a9bHHdfp7FQjZbD0Vbr0ran9EBo/copy
前端、後端程式已含

**2. 開啟 Apps Script**

在試算表中點選「擴充功能」→「Apps Script」，進入編輯器。

**3. 部署為 Web App**

- 點選「部署」→「新增部署」→ 類型選「網頁應用程式」。
- 執行身分：**我**（以部署者身分執行）。
- 存取權限：依組織需求選擇（建議「組織內所有人」）。
- 複製產生的 URL 分享給使用者。

## 資料庫結構

### QDP_Settings

| 欄位 | 說明 |
|---|---|
| 活動名稱 | 活動唯一代號，例如 `2026-2` |
| 參與單位 | 以逗號分隔，例如 `研發部,業務部,品管部` |
| 最低提案數 | 每單位提案下限（數字） |
| 每人總票數 | 每位投票者需投出的票數（數字） |
| 當前階段 | `準備中` / `提案階段` / `投票階段` / `評估對策` / `完全關閉` |

### QDP_Proposals

| 欄位 | 說明 |
|---|---|
| 提案ID | 系統自動生成，格式 `P{timestamp}-{index}` |
| 活動名稱 | 對應 QDP_Settings 活動代號 |
| 提案單位 | 提案所屬部門 |
| 提案人 | 提案人姓名 |
| 關注議題 | 議題內容 |

### QDP_Votes

| 欄位 | 說明 |
|---|---|
| 投票ID | 系統自動生成 |
| 活動名稱 | 對應活動代號 |
| 投票單位與人名 | 格式 `部門-姓名` |
| 匿名代號 | 格式 `#1`、`#2`，同人重複投票沿用舊代號 |
| 目標提案ID | 對應 QDP_Proposals 的提案ID |
| 給予分數 | 數字 |

### QDP_ActionPlans

| 欄位 | 說明 |
|---|---|
| 活動名稱 | 對應活動代號 |
| 提案ID | 對應提案 |
| 統計總得分 | 保留欄位（目前由系統即時計算，未寫入） |
| 投票人數 | 保留欄位（目前由系統即時計算，未寫入） |
| 管理員優先次序 | 管理員手動排定的數字順序 |
| 可能影響 | 對策表填寫欄位 |
| 負責部門 | 對策表填寫欄位 |
| 現行管制 | 對策表填寫欄位 |
| 應對措施 | 對策表填寫欄位 |
| 預計完成時間 | 對策表填寫欄位 |
| 內部/外部 | 對策表填寫欄位 |
| 利害相關人 | 對策表填寫欄位 |

## 使用說明

### 一般使用者

1. 開啟管理員提供的 Web App URL。
2. 在「活動大廳」點選開放中的活動進入。
3. 依系統引導完成提案（S1）→ 投票（S2）流程。
4. 評估紀錄（S3）與對策表（S4）為唯讀模式。

### 管理員

1. 點選左下角「管理員設定」，輸入密碼登入。
2. 在管理員頁面新增活動、設定參與單位與規則。
3. 依進度切換活動階段（提案 → 投票 → 評估對策）。
4. 登入後可在評估紀錄（S3）設定優先次序，在對策表（S4）填寫應對措施。
5. 刪除活動時，系統會同步清除四個 Sheet 中的所有相關資料。

## 安全性說明

| 項目 | 說明 |
|---|---|
| 管理員密碼 | 硬編碼於 `QDP_backend.gs` 的 `ADMIN_PASSWORD` 變數，部署前請修改預設值 `'admin'` |
| 匿名機制 | 前端評估紀錄表僅顯示 `#編號`，真實姓名保留在後端 Sheet |
| 並發保護 | 投票寫入使用 `LockService.getScriptLock()` 防止競態 |
| 存取控制 | 依 GAS 部署設定決定，建議限制為組織內部成員 |

> ⚠️ **注意**：GAS Web App 本身無法對單一函式設定速率限制，管理員密碼建議在部署後立即修改為複雜字串。

## 已知限制

- GAS 單次執行上限為 **6 分鐘**，大量資料（數百列以上）的刪除操作可能逾時。
- GAS 免費版每日執行配額有限，高頻率使用建議評估 Workspace 方案。
- 試算表欄位請勿手動新增或刪除，否則可能造成欄位索引錯位。
- `QDP_ActionPlans` 的第3、4欄（統計總得分、投票人數）為保留欄位，目前由系統即時計算，不寫入試算表。
